# 仓库重构方案

## 背景

当前仓库结构存在以下问题：
- **c++/ 根目录扁平化**：263 个文件混在一起，语法学习代码、算法模板、散题全部同级
- **c++/练习/ 大量冗余**：200+ 文件中有大量空文件和无意义的 "copy N" 备份
- **学习路径不清晰**：C++ 语法学习（0-116 编号系列）与 AcWing 算法模板穿插但无明确分界
- **目录命名不统一**：中英文混用、无层次感

## 执行步骤

### Step 1: 删除空文件

**约 100+ 个空文件**，分为几类：

| 类别 | 数量 | 示例 |
|------|------|------|
| c++/根目录空.txt | 3 | `0-1C++入门.txt`, `2-8数据类型.txt`, `9-14运算符.txt` |
| c++/根目录空.cpp | 1 | `零时.cpp` |
| c++/练习/自律 copy N | 50+ | `自律 copy 6.cpp` ~ `自律 copy 67.cpp`（空的副本） |
| c++/练习/超级多的练习-副本 | 12 | 空的 |
| c++/练习/其他 | 2 | `------项目------.cpp`, `李世海-实验二 copy 3.cpp` |
| 算法/空.cpp | ~30 | 各比赛目录下未实现的题解空文件 |
| 算法/空.md | ~10 | 未写完的题解空 markdown |
| markdown/空 | 2 | `m2总结.md`, `临时.md` |

### Step 2: 重构 cpp/ 目录

**核心思路**：按学习阶段将 263 个文件分为 6 个子目录。

#### 第一阶段：01-basics（约 40 个文件）
- **变量/常量/数据类型**: `2.0变量.cpp`, `2.1浮点型变量.cpp`, `3.常量.cpp`, `5.字符型.cpp`, `6.字符串.cpp`, `7.bool.cpp`, `8.输入.cpp`
- **运算符**: `9~14+` 系列（加减乘除、取模、递增递减、赋值、比较、逻辑、按位与）
- **流程控制**: `15~31` 系列（if、switch、while、for、break/continue/goto、猜数字、水仙花数、敲桌子、乘法口诀表）
- **数组**: `32~39` 系列（一维数组、二维数组、冒泡排序、成绩统计）
- 配套的 `.txt` 笔记文件（15-31程序流程结构.txt、32-39数组.txt）也移入

#### 第二阶段：02-pointer-function（约 20 个文件）
- **函数**: `40~45` 系列 + `72~74`（函数高级：默认参数、占位参数、重载）
- **指针**: `46~52` 系列（定义、空指针、const修饰、指针+数组+函数案例）
- **结构体**: `53~60` 系列（定义、嵌套、函数参数、const、案例）

#### 第三阶段：03-memory-reference（约 10 个文件）
- **内存模型**: `61~65`（全局区、栈区、堆区、new）
- **引用**: `66~71`（语法、注意事项、引用做参数/返回值、本质、常量引用）

#### 第四阶段：04-oop（约 50 个文件）
- **封装**: `75~80+` 系列（学生类、访问权限、立方体案例、点和圆）
- **构造析构**: `81~88` 系列（构造分类、拷贝、深拷贝浅拷贝、初始化列表、静态成员）
- **this指针**: `89~92`
- **友元**: `93~95`
- **运算符重载**: `96~101`
- **继承**: `102~109`（方式、对象模型、构造析构顺序、同名成员、多继承、菱形继承）
- **多态**: `110~116`（基本概念、计算器类、纯虚函数、制作饮品、虚析构、电脑组装案例）
- **STL**: `STL算法-1~7` 系列
- **复盘**: `复盘_1~13` 系列

#### 第五阶段：05-algorithms（约 35 个文件）
- AcWing 算法模板 `acwing- 1~33`，**按主题进一步组织**：
  - **排序**: 快排、归并排序
  - **高精度**: 加/减/乘/除
  - **数据结构**: 前缀和、差分、双指针、位运算、离散化、区间合并、链表、栈、队列、单调栈/队列、KMP、Trie、并查集、堆、哈希表
  - **图论**: DFS、BFS、拓扑排序、最短路（Dijkstra/Bellman-Ford/SPFA/Floyd）、最小生成树（Prim/Kruskal）、二分图（染色法/匈牙利）
- 散落在根目录的独立算法文件：快速幂、大整数类、选择排序等

#### 第六阶段：06-practice（约 20 个有意义的练习文件）
从 c++/练习/ 中筛选出有意义的文件：
- `刷题-洛谷*`, `刷题-牛客*`, `刷题-acwing*`, `刷题-codeforces*`
- `手搓-1.*`（去重后保留一份）
- `练习2`, `循环练习`, `自律.cpp`（非空的主文件）
- `暴力移除数组元素.cpp`, `高效判断质数.cpp`, `状态压缩dp.cpp` 等

#### 07-exercises（课程作业）
- `李世海-实验*.cpp` 系列（非空且保留）

#### archive/（有内容但重复的备份文件）
- 有内容的 "copy" 文件但基本与主版本重复 → 移入 archive/

### Step 3: 重构 algorithms/ 目录

**核心思路**：保留按比赛分组的逻辑，但统一命名规范，合并散落的散题。

1. **目录重命名**（简化长目录名）：
   - `Codeforces Round 1045 (Div. 2)` → `CF-1045-Div2`
   - `牛客周赛 Round 127` → `Nowcoder-127`
   - `2025年香港城市大学（东莞）新生程序设计个人排位赛正式赛` → `CityU-DG-2025`
   - `The 2024 CCPC National Invitational Contest (Changchun)` → `CCPC-2024-Changchun`
   - 以此类推

2. **tools/** 子目录：
   - 将 `翻译姬*.md`, `根据cpp文件批量创建md工具*` 移入

3. **topics/** 子目录：
   - 将原来 `散题/` 下的按算法主题分类（如果可分类的话）

4. **清理空文件**：
   - 删除所有空的 `.cpp` 和 `.md` 文件

### Step 4: 重构 notes/ 目录

合并三个目录：
- `MySQL/` → `notes/mysql/`
- `理论/操作系统概论整理/` → `notes/os/`
- `markdown/linux指令.md` → `notes/linux.md`
- `markdown/m1题目模板.md` 等模板 → `notes/templates/`

### Step 5: 重构 personal/ 目录

从原 `markdown/` 中分离个人内容：
- 日记 `日记2025-*.md` → `personal/diary/`
- 计划 `计划*.md` → `personal/plans/`
- `刷题.md`, `日常推送.md` → 视内容决定去向

### Step 6: 更新 CLAUDE.md 和 INDEX.md

- 更新 `CLAUDE.md` 反映新结构
- 完全重写 `INDEX.md` 反映新的目录结构

## 验证

1. 重构后检查所有 `.cpp` 文件是否能正常打开（内容未丢失）
2. 检查 INDEX.md 中所有链接是否正确
3. 执行 `git status` 确认文件变动符合预期
