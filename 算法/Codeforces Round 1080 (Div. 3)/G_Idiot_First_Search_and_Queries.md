# **【算法题】G. Idiot First Search and Queries（傻瓜优先搜索与查询）**
每测试用例时间限制：4 秒
每测试用例内存限制：512 兆字节

### 题目描述
本题与题目E共享一些定义，但所求答案不同。

有一棵 \(n + 1\) 个顶点的二叉树（\(n\) 为奇数），顶点标记为 \(0, 1, \cdots, n\) 。树上每个顶点最多可写一个字母，且初始时所有顶点均无字母。树的根节点是顶点 \(0\) 。

在这棵树中，顶点 \(0\) 是顶点 \(1\) 的父节点，而其他所有顶点要么有 \(2\) 个子节点，要么有 \(0\) 个子节点。

鲍勃迷失在树的某个顶点上，他希望通过到达顶点 \(0\) 来逃离这棵树。对于大多数有常识的人来说，这非常容易。然而，由于鲍勃是个傻瓜，他创造了一种新的遍历树的方法，即 “傻瓜优先搜索”。

当鲍勃在顶点 \(v\)（\(1 \leq v \leq n\)）时，他的移动方式如下：
 - 如果顶点 \(v\) 是叶子节点，鲍勃总是移动到 \(v\) 的父节点；否则，检查接下来的几个条件。
 - 如果顶点 \(v\) 上没有写任何东西，鲍勃在顶点 \(v\) 上写 “L” 并移动到 \(v\) 的左子节点；
 - 如果顶点 \(v\) 上写着 “L”，鲍勃将其覆盖为 “R” 并移动到 \(v\) 的右子节点；
 - 如果顶点 \(v\) 上写着 “R”，鲍勃擦掉它并移动到 \(v\) 的父节点。

鲍勃移动到相邻顶点恰好需要 \(1\) 秒，所以鲍勃进行 \(x\) 次移动恰好需要 \(x\) 秒。

已经证明，无论鲍勃从哪个顶点出发，他都能在有限（尽管可能长得离谱）的时间内到达顶点 \(0\) 。我们不知道是谁证明了这一点；肯定不是鲍勃，但这确实已经被证明了。

你需要回答 \(q\) 个以下类型的查询：
\(v_k\) ：假设鲍勃从顶点 \(v\) 出发，确定鲍勃在恰好进行 \(k\) 次移动后所在的顶点（\(1 \leq v \leq n\)）。
对于每个查询，设 \(T_v\) 为从顶点 \(v\) 到达顶点 \(0\) 所需的时间。那么，保证每个查询中的 \(k < T_v\) 。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 \(t\)（\(1 \leq t \leq 10^4\)）。接下来是测试用例的描述。

每个测试用例的第一行包含整数 \(n\) 和 \(q\)（\(1 \leq n \leq 300001\)，\(1 \leq q \leq 400000\)，\(n\) 为奇数）。

接下来 \(n\) 行，每行包含两个整数 \(l_i\) 和 \(r_i\)，表示顶点 \(i\) 的子节点（\(0 \leq l_i, r_i \leq n\)）。

对于每个顶点，如果该顶点没有子节点，则 \(l_i = r_i = 0\) 。否则，\(l_i\) 和 \(r_i\) 分别是顶点 \(i\) 的左子节点和右子节点。

接下来 \(q\) 行，每行包含两个整数 \(v_j\) 和 \(k_j\)，表示第 \(j\) 个查询（\(1 \leq v_j \leq n\)，\(0 \leq k_j < \min(10^9 + 7, T_{v_j})\)）。

保证输入定义了一棵满足题目给定条件的有效二叉树。

保证所有测试用例的 \(n\) 之和不超过 \(300001\) 。

保证所有测试用例的 \(q\) 之和不超过 \(400000\) 。

### 输出
对于每个测试用例，在单独一行输出 \(q\) 个查询的答案。

### 说明
在第一个测试用例中，只有两个顶点，顶点 \(0\) 和顶点 \(1\) 。显然，鲍勃从顶点 \(1\) 出发，在进行 \(0\) 次移动后，他将在顶点 \(1\) 。

在第二个测试用例中，树的结构如下。

鲍勃从顶点 \(3\) 到达顶点 \(0\) 需要 \(14\) 秒。移动过程如下：
\(3 \xrightarrow{L} 4 \xrightarrow{X} 3 \xrightarrow{R} 5 \xrightarrow{X} 3 \xrightarrow{X} 1 \xrightarrow{L} 2 \xrightarrow{X} 1 \xrightarrow{R} 3 \xrightarrow{L} 4 \xrightarrow{X} 3 \xrightarrow{R} 5 \xrightarrow{X} 3 \xrightarrow{X} 1 \xrightarrow{X} 0\)

这里，箭头上方的字母表示移动到相邻顶点之前该顶点上的字母，其中 \(X\) 表示没有写任何东西。
