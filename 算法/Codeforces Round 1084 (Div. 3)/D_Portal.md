# **【算法题】D. Portal（传送门）**
每测试用例时间限制：2 秒
每测试用例内存限制：256 兆字节

### 题目描述
给定一个长度为 \(n\) 的排列\(^{*}\) \(p\) 。同时有两个传送门，分别位于位置 \(x\) 和 \(y\)（\(x < y\)）。

位于位置 \(i\) 的传送门最初位于数组的第 \(i\) 个元素和第 \((i + 1)\) 个元素之间。特别地，如果 \(i = 0\)，那么传送门位于数组的第一个元素之前；如果 \(i = n\)，那么传送门位于数组的最后一个元素之后。

你可以根据需要多次执行以下两种操作之一：
- 移除一个传送门紧邻左侧的元素，并将其插入到另一个传送门紧邻右侧。
- 移除一个传送门紧邻右侧的元素，并将其插入到另一个传送门紧邻左侧。

设 \(\textit{\textbf{O}}\) 表示传送门。例如，如果 \(p\) 是 \([3, \textit{\textbf{O}}, 2, 4, \textit{\textbf{O}}, 1]\)：
 - 分别对左右传送门使用操作1，得到数组 \([\textit{\textbf{O}}, 2, 4, \textit{\textbf{O}}, 3, 1]\) 和 \([3, \textit{\textbf{O}}, 4, 2, \textit{\textbf{O}}, 1]\) 。
 - 分别对左右传送门使用操作2，得到数组 \([3, \textit{\textbf{O}}, 4, 2, \textit{\textbf{O}}, 1]\) 和 \([3, 1, \textit{\textbf{O}}, 2, 4, \textit{\textbf{O}}]\) 。

找出通过这些操作能得到的字典序\(^{\dagger}\)最小的排列。注意，传送门不影响排列的字典序比较。

### 名词解释
 - \(^{*}\) 长度为 \(n\) 的排列是一个长度为 \(n\) 的数组，其中包含从 \(1\) 到 \(n\) 的每个整数恰好一次。
 - \(^{\dagger}\) 排列 \(a\) 在字典序上小于排列 \(b\)，如果存在一个索引 \(i\)，使得对于所有 \(1 \leq j < i\) 的索引，都有 \(a_j = b_j\)，且 \(a_i < b_i\) 。

### 输入
第一行包含一个整数 \(t\)（\(1 \leq t \leq 2 \cdot 10^4\)）—— 测试用例的数量。

对于每个测试用例，第一行包含三个整数 \(n\)、\(x\) 和 \(y\)（\(1 \leq n \leq 2 \cdot 10^5\)，\(0 \leq x < y \leq n\)）。

每个测试用例的第二行包含 \(n\) 个整数 \(p_1, p_2, \cdots, p_n\) —— 一个长度为 \(n\) 的排列。

所有测试用例的 \(n\) 之和不超过 \(2 \cdot 10^5\) 。

### 输出
对于每个测试用例，在一行中输出 \(n\) 个整数 —— 你能得到的字典序最小的排列。

### 示例
- **输入**
```
4
4 0 4
3 1 4 2
3 1 2
3 2 1
5 1 3
1 3 5 2 4
2 0 1
1 2
```
- **输出**
```
1 4 2 3
2 3 1
1 2 3 5 4
1 2
```

### 说明
设 \(\textit{\textbf{O}}\) 表示传送门。

在第一个测试用例中，数组是 \([\textit{\textbf{O}}, 3, 1, 4, 2, \textit{\textbf{O}}]\) 。对左传送门使用操作2得到 \([\textit{\textbf{O}}, 1, 4, 2, 3, \textit{\textbf{O}}]\) ，这是能得到的字典序最小的排列。

![操作描述](./D_Portal_1.gif)

在第二个测试用例中，数组是 \([3, \textit{\textbf{O}}, 2, \textit{\textbf{O}}, 1]\) 。对左传送门使用操作1得到 \([\textit{\textbf{O}}, 2, \textit{\textbf{O}}, 3, 1]\) ，这是能得到的字典序最小的排列。

在第四个测试用例中，不进行任何操作是最优的。
