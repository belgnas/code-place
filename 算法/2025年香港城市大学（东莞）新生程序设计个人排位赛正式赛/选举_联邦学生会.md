# **【算法题】选举：联邦学生会**

时间限制：C/C++/Rust/Pascal 3秒，其他语言6秒
空间限制：C/C++/Rust/Pascal 256 M，其他语言512 M
64bit IO Format: %lld

### 题目描述
联邦学生会即将迎来新一轮的选举！

有 $n$ 个人报名参加了新一届联邦学生会的选举，每个候选人都有一个能力值。

在选举的一开始，由于所有的学生都太喜欢 `sensei` 了，所以会先全票通过 `sensei` 成为新的联邦学生会主席。接下来，就会由 `sensei` 来选择任意名他喜欢的学生，使其也能被选举进入联邦学生会。

然而，`sensei` 不希望能力值与他相当或者能力值比他大的学生加入联邦学生会，因为这会导致他的领导地位受到动摇。同时，`sensei` 又希望整个联邦学生会所有成员的总能力值尽可能的大。

你可以帮助 `sensei` 求出这个最大的可能的总能力值吗？

### 输入描述
输入的第一行包含一个整数 $n$（$1\le n\le10^{5}$），表示报名参加新一届联邦学生会选举的总人数（含 `sensei` 自己）。

接下来输入 $n$ 行，第 $i$ 行包含一个仅由小写字母构成的字符串 $s_{i}$（$1\le|s_{i}|\le10^{6}$） 和一个正整数 $a_{i}$（$1\le a_{i}\le10^{9}$），分别表示第 $i$ 个候选人的名字和能力值。

数据保证所有候选人的名字均唯一，同时有且仅有一个候选人的名字为 `sensei`，所有人名字符串长度的总和 $\sum_{i = 1}^{n}|s_{i}|$ 不超过 $10^{6}$。

### 输出描述
输出一行一个正整数，表示最终最大的可能的整个联邦学生会中所有成员的总能力值。

### 示例1
- **输入**
```
4
sensei 89
serika 64
hoshino 114
shiroko 514
```
- **输出**
```
153
```
- **说明**
`sensei` 只能选择 `serika` 加入联邦学生会，最终整个联邦学生会所有成员总能力值为 $89+64=153$。可以证明这是所有可行方案中的最大值。
