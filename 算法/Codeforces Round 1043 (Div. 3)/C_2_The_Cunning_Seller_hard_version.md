# 【算法题】The Cunning Seller (hard version)（狡猾的卖家（困难版））

**时间限制**：每个测试用例2秒  
**内存限制**：每个测试用例256兆字节  

这是该问题的困难版本。简单版本与困难版本的不同之处在于：简单版本要求确定“交易次数最少”时的最小成本，而困难版本要求在“有限交易次数（不超过$k$次）”下确定最小成本。


在狡猾的卖家以“1个西瓜的价格卖3个西瓜”后，他决定进一步提高利润——于是购买了更多西瓜。现在，他的销售规则如下：  
可以以$3^{x + 1} + x \cdot 3^{x - 1}$枚硬币的价格（其中$x$是非负整数），卖出$3^x$个西瓜。这样的一次销售称为**一笔交易**。

一个精明的买家来找他，但时间有限：买家最多只能进行$k$笔交易，且计划**正好购买$n$个西瓜**。  

买家急需帮助，请求你计算：若他进行不超过$k$笔交易，购买$n$个西瓜所需支付的**最少硬币数**。若在不超过$k$笔交易的情况下无法正好购买$n$个西瓜，则输出$-1$。


## 输入
第一行包含一个整数$t$（$1 \leq t \leq 10^4$）——测试用例的数量。  
每个测试用例占一行，包含两个整数$n$和$k$（$1 \leq n,k \leq 10^9$）——分别表示“需要购买的西瓜总数”和“最多可进行的交易次数”。


## 输出
对于每个测试用例，输出一个整数：  
- 若能满足条件（不超过$k$笔交易、正好买$n$个西瓜），则输出最小成本；  
- 否则输出$-1$。


## 示例
### 输入
```
8
1 1
3 3
8 3
2 4
10 10
20 14
3 2
9 1
```

### 输出
```
3
9
-1
6
30
63
10
33
```


## 注意
- 购买比所需数量多的西瓜无意义，因此不考虑西瓜数量超过$n$的交易组合。  
- 基础交易的成本验证（结合示例）：  
  - 当$x=0$时（“交易A”）：卖出$3^0 = 1$个西瓜，价格为$3^0 + 1 + 0 \cdot 3^{-1} = 1 + 1 + 0 = 3$枚硬币（对应示例1：1个西瓜用1笔“交易A”，成本为3）。  
  - 当$x=1$时（“交易B”）：卖出$3^1 = 3$个西瓜，价格需以示例为准（示例2中，1笔“交易B”买3个西瓜成本为10，而3笔“交易A”买3个西瓜成本为$3 \times 3 = 9$，因此更优选择是3笔“交易A”，成本为9）。  
- 示例3解释：3笔交易的所有可能组合都无法凑出8个西瓜：  
  - 3笔“交易A”：$1 \times 3 = 3$个西瓜；  
  - 2笔“交易A” + 1笔“交易B”：$1 \times 2 + 3 \times 1 = 5$个西瓜；  
  - 1笔“交易A” + 2笔“交易B”：$1 \times 1 + 3 \times 2 = 7$个西瓜；  
  - 3笔“交易B”：$3 \times 3 = 9$个西瓜；  
  因此输出$-1$。
