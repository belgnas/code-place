# **【算法题】D. Christmas Tree Decoration（圣诞树装饰）**
每测试用例时间限制：2 秒
每测试用例内存限制：512 兆字节

### 题目描述
有 \(n\) 个人决定装饰圣诞树。他们有 \((n + 1)\) 个装饰盒，编号从 \(0\) 到 \(n\) 。最初，第 \(i\) 个盒子里有 \(a_i\) 个装饰品。

我们称一个大小为 \(n\) 的排列 \(p\)（一个大小为 \(n\) 的数组，其中从 \(1\) 到 \(n\) 的每个数字恰好出现一次）是公平的，如果可以使用以下过程将所有装饰品挂到树上：
 - 人 \(p_1\) 从盒子 \(0\) 或盒子 \(p_1\) 中取出一个装饰品，并挂到树上；
 - 人 \(p_2\) 从盒子 \(0\) 或盒子 \(p_2\) 中取出一个装饰品，并挂到树上；
 - 依此类推；
 - 在人 \(p_n\) 之后，人 \(p_1\) 接着进行，这个过程重复，直到所有装饰品都挂到树上。

在这个过程中，不应出现人 \(i\) 需要挂装饰品，但盒子 \(0\) 和盒子 \(i\) 都为空的情况。如果无法避免这种情况，那么这个排列就不公平。然而，如果人们在每一步都能选择从哪个盒子取装饰品，使得这种情况不会出现，那么这个排列就是公平的。

你的任务是计算公平排列的数量。由于答案可能很大，输出结果对 \(998244353\) 取模。

### 输入
第一行包含一个整数 \(t\)（\(1 \leq t \leq 5000\)）—— 测试用例的数量。

每个测试用例的第一行包含一个整数 \(n\)（\(1 \leq n \leq 50\)）。

第二行包含 \((n + 1)\) 个整数 \(a_0, a_1, \cdots, a_n\)（\(0 \leq a_i \leq 10^6\)）。

### 输出
对于每个测试用例，输出一个整数 —— 公平排列的数量，对 \(998244353\) 取模。

### 示例
- **输入**
```
4
3
1 2 1 0
3
1 0 2 0
1
2 5
4
6 1 4 2 1
```
- **输出**
```
2
0
1
12
```

### 说明
在第一个示例中，公平排列是 \([1, 2, 3]\) 和 \([1, 3, 2]\) 。

让我们仔细看看排列 \([1, 3, 2]\) 的圣诞树装饰过程：
 - 人 \(p_1 = 1\) 从盒子 \(1\) 中挂一个装饰品；
 - 人 \(p_2 = 3\) 从盒子 \(0\) 中挂一个装饰品；
 - 人 \(p_3 = 2\) 从盒子 \(2\) 中挂一个装饰品；
 - 人 \(p_1 = 1\) 从盒子 \(1\) 中挂一个装饰品。

注意，如果人 \(p_1 = 1\) 在第一步从盒子 \(0\) 中取装饰品，那么人 \(p_2 = 3\) 将无法进行下一步（因为盒子 \(0\) 和盒子 \(3\) 都将为空）。然而，由于这种情况可以避免，所以这个排列是公平的。
