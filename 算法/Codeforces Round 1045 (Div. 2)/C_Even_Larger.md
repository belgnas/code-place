# **【算法题】C. Even Larger（偶数位置和更大）**
- **每个测试的时间限制**：2 秒
- **每个测试的内存限制**：256 兆字节

如果对于长度至少为 2 的每个子数组∗，偶数索引（相对于原始数组）处元素的和大于或等于奇数索引处元素的和，则称该数组为“好数组”。数组索引从 1 开始。

例如，数组 $[3, 8, 4, 4]$ 和 $[2, 3, 1, 4, 2]$ 是好数组。数组 $[0, 2, 4, 1]$ 不是好数组，因为在子数组 $[2, 4, 1]$ 中，原始数组中偶数索引处的元素是 2（索引 2）和 1（索引 4），而奇数索引处唯一的元素是 4（索引 3）。由于 $2 + 1 < 4$，该子数组不满足条件。

给定一个由 $n$ 个非负整数组成的数组 $a_1, a_2, \ldots, a_n$。在一次操作中，你可以将数组中的任何元素减 1，但所有元素必须保持非负。你的任务是找到使数组 $a$ 成为好数组所需的最少操作次数。可以证明，使用有限次操作可以使数组变为好数组。

∗ 如果数组 $b$ 可以通过从数组 $a$ 的开头删除若干（可能为零个或全部）元素以及从结尾删除若干（可能为零个或全部）元素而得到，则称数组 $b$ 是数组 $a$ 的子数组。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 $t$（$1 \leq t \leq 10^4$）。接下来是测试用例的描述。

每个测试用例的第一行包含一个整数 $n$（$2 \leq n \leq 2 \cdot 10^5$）—— 数组 $a$ 的长度。

每个测试用例的第二行包含 $n$ 个非负整数 $a_1, a_2, \ldots, a_n$（$0 \leq a_i \leq 10^9$）—— 数组 $a$ 的元素。

保证所有测试用例中 $n$ 的总和不超过 $2 \cdot 10^5$。

### 输出
对于每个测试用例，在新的一行中输出一个整数 —— 使数组 $a$ 成为好数组所需的最少操作次数。

### 注意
在第一个测试用例中，数组 $a$ 已经是好数组，所以答案是 0。以下是对每个子数组的检查：

|子数组|偶数索引和|奇数索引和|条件满足吗？|
| ---- | ---- | ---- | ---- |
|$[3, 8]$|8|3|是|
|$[8, 4]$|8|4|是|
|$[4, 4]$|4|4|是|
|$[3, 8, 4]$|8|$3 + 4 = 7$|是|
|$[8, 4, 4]$|$8 + 4 = 12$|4|是|
|$[3, 8, 4, 4]$|$8 + 4 = 12$|$3 + 4 = 7$|是|

在第二个测试用例中，数组 $a$ 不是好数组：

|子数组|偶数索引和|奇数索引和|条件满足吗？|
| ---- | ---- | ---- | ---- |
|$[0, 2]$|2|0|是|
|$[2, 3]$|2|3|否|
|$[3, 5]$|5|3|是|
|$[0, 2, 3]$|2|$0 + 3 = 3$|否|
|$[2, 3, 5]$|$2 + 5 = 7$|3|是|
|$[0, 2, 3, 5]$|$2 + 5 = 7$|$0 + 3 = 3$|是|

然而，如果我们对索引 3 执行一次操作，数组变为 $[0, 2, 2, 5]$，现在是好数组了：

|子数组|偶数索引和|奇数索引和|条件满足吗？|
| ---- | ---- | ---- | ---- |
|$[0, 2]$|2|0|是|
|$[2, 2]$|2|2|是|
|$[2, 5]$|5|2|是|
|$[0, 2, 2]$|2|$0 + 2 = 2$|是|
|$[2, 2, 5]$|$2 + 5 = 7$|2|是|
|$[0, 2, 2, 5]$|$2 + 5 = 7$|$0 + 2 = 2$|是|

因此，答案是 1。
