# **【算法题】A. Blackslex and Password（布莱克斯莱克斯与密码）**

每个测试的时间限制：1 秒
每个测试的内存限制：256 兆字节

布莱克斯莱克斯正在为吉恩·德夫设计一个登录系统，他发现大多数用户使用的密码都很弱。

为解决这个问题，他针对所有密码，依据两个变量\(k\)和\(x\)，提出了以下条件。每个密码都是一个长度为\(n\)的字符串\(s\)，需满足这些属性：
1. \(s\)仅使用英文字母表的前\(k\)个小写字母。
2. 对于每一对索引\(1 \leq i < j \leq n\) ，若\((j - i)\)能被\(x\)整除，则字母\(s_i\)和\(s_j\)不同。

找出不存在有效字符串的最小整数\(n\) 。

### 输入
第一行包含一个整数\(t\)（\(1 \leq t \leq 500\)）——测试用例的数量。

每个测试用例的第一行也是唯一一行，包含两个整数\(k\)和\(x\)（\(1 \leq k \leq 26\)，\(1 \leq x \leq 15\)）。

### 输出
对于每个测试用例，输出最小的\(n\) 。

### 示例
- **输入**
```
3
2 1
3 2
1 5
```
- **输出**
```
3
7
6
```
### 说明
对于第一个测试用例，不存在长度\(n = 3\)的有效字符串。对于\(n = 2\) ，一个这样的有效示例是“ab”。注意，对于\(n = 2\) ，唯一满足\((j - i)\)能被\(x = 1\)整除且\(1 \leq i < j \leq n\)的对\((i, j)\)是\((1, 2)\) 。

对于第二个测试用例，不存在长度\(n = 7\)的有效字符串。对于\(n = 6\) ，一个这样的有效示例是“aabccb”。注意，对于\(n = 6\) ，所有满足\((j - i)\)能被\(x = 2\)整除且\(1 \leq i < j \leq n\)的对\((i, j)\)包括\((1, 3)\) 、\((1, 5)\) 、\((2, 4)\) 、\((2, 6)\) 、\((3, 5)\)和\((4, 6)\) 。

对于第三个测试用例，不存在长度\(n = 6\)的有效字符串。对于\(n = 5\) ，一个这样的有效示例是“aaaaa”。注意，对于\(n = 5\) ，不存在满足\((j - i)\)能被\(x = 5\)整除且\(1 \leq i < j \leq n\)的对\((i, j)\) 。
