# **【算法题】B. OIE Excursion（OIE远足）**
每个测试用例的时间限制：1 秒
每个测试用例的内存限制：256 兆字节

赫克托正在参加西班牙信息学奥林匹克的远足活动，游览拉科鲁尼亚，但他迫切想偷偷溜走，去见他的朋友古斯塔沃、埃索默和达尼。为此，他必须穿过一条由 \(n\) 名志愿者看守的小路，这些志愿者排成一排，从 \(1\) 到 \(n\) 编号；第 \(i\) 名志愿者看守位置 \(i\)。

每个志愿者都有一个内部计时器。最初（在第 \(0\) 秒），第 \(i\) 名志愿者的计时器值为 \(a_i\)。每秒，所有计时器的值都增加 \(1\)。一旦计时器达到 \(m\)，它就会回绕到 \(0\)。具体来说，在第 \(x\) 秒，第 \(i\) 名志愿者的计时器显示的值为 \((a_i + x) \bmod m\)。

只有当第 \(i\) 名志愿者的计时器恰好为 \(0\) 时，他才会看守位置 \(i\)。在其他任何时候，他们都会分心，不会注意到赫克托。

赫克托从位置 \(0\) 出发，在所有志愿者的左边。为了逃脱，他必须到达位置 \(n + 1\)。在每一秒结束时，赫克托可以选择留在当前位置，向左移动一个位置，或者向右移动一个位置。请注意，赫克托不能移动到位置 \(0\) 的左边。

当且仅当在某一秒开始时，赫克托处于位置 \(i\)（\(1 \leq i \leq n\)）且第 \(i\) 名志愿者的计时器为 \(0\) 时，赫克托才会被抓住。

判断是否存在一种策略，让赫克托能够穿过所有志愿者并逃脱而不被抓住。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 \(t\)（\(1 \leq t \leq 10^4\)）。接下来是测试用例的描述。

每个测试用例由两行组成：
- 第一行包含两个整数 \(n\) 和 \(m\)（\(2 \leq n \leq 2 \cdot 10^5\)，\(2 \leq m \leq 10^9\)）——志愿者的数量和计时器的循环长度。
- 第二行包含 \(n\) 个整数 \(a_1, a_2, \ldots, a_n\)（\(0 \leq a_i < m\)），其中 \(a_i\) 表示在 \(t = 0\) 时第 \(i\) 名志愿者计时器的初始值。

保证所有测试用例中 \(n\) 的总和不超过 \(2 \cdot 10^5\)。

### 输出
对于每个测试用例，打印一行，包含 “YES” 或 “NO”，表示赫克托是否能够逃脱。答案不区分大小写。例如，字符串 “yEs”、“yes”、“Yes” 和 “YES” 都将被识别为肯定回答。

### 示例
**输入**
```
6
8 5
0 4 0 2 1 0 0 3
6 2
1 0 1 0 1 0
6 2
1 1 1 1 0 1
2 10
6 9
2 2
0 1
5 1000000000
1 2 3 4 999999999
```
**输出**
```
YES
YES
NO
YES
YES
YES
```

### 说明
- 案例1：赫克托可以每秒都向右移动，无需等待或向左移动就能逃脱，因为不存在 \(i\) 使得 \((a_i + i) \bmod m = 0\)。
- 案例2：一种可能的策略是在起始位置等待一秒，然后每秒都向右移动，不再等待或向左移动。
