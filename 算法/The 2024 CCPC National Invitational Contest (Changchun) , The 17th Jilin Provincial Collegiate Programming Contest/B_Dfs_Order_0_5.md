# **【算法题】Dfs Order 0.5（深度优先搜索顺序0.5）**
**输入文件**：标准输入
**输出文件**：标准输出
**时间限制**：1 秒
**内存限制**：512 兆字节

水水有一棵有 \(n\) 个顶点的树，顶点编号从 \(1\) 到 \(n\)，其中顶点 \(1\) 是根节点。第 \(i\) 个顶点的值为 \(a_i\) 。

现在从根节点开始进行深度优先搜索。由于一个节点的子节点可以以任意顺序遍历，所以存在多种可能的深度优先搜索顺序。我们将深度优先搜索顺序的值定义为在深度优先搜索顺序中出现在偶数索引位置的顶点值之和。水水想知道在给定树的所有可能的深度优先搜索顺序中，最大值是多少。

以下是在有根树上进行深度优先搜索的伪代码。调用 `main()` 后，`dfs_order` 将是深度优先搜索顺序。

**算法1：深度优先搜索的实现**
```
1: procedure dfs(vertex x)
2:     Append x to the end of dfs_order
3:     for each son y of x do
4:         dfs(y)
5: procedure main()
6:     Let dfs_order be a global variable
7:     dfs_order ← empty list
8:     dfs(1)
```
子节点可以以任意顺序遍历。

以下是计算深度优先搜索顺序值的伪代码。调用 `calc(dfs_order)` 将返回其值。

**算法2：计算深度优先搜索顺序值的实现**
```
1: procedure calc(dfs_order)
2:     // dfs_order 是一个基于1索引、长度为 n 的数组
3:     s ← 0
4:     p ← 2
5:     while p ≤ n do
6:         u ← dfs_order[p]
7:         s ← s + a[u]
8:         p ← p + 2
9:     return s
```

### 输入
输入包含多个测试用例。
第一行包含一个整数 \(t\)（\(1 \leq t \leq 2×10^5\)），表示测试用例的数量。
对于每个测试用例：
 - 第一行包含一个整数 \(n\)（\(1 \leq n \leq 2×10^5\)），表示树中顶点的数量。
 - 第二行包含 \(n\) 个整数 \(a_1, a_2, \cdots, a_n\)（\(1 \leq a_i \leq 10^9\)），表示这些顶点的值。
 - 接下来 \(n - 1\) 行，每行包含两个整数 \(u_i, v_i\)（\(1 \leq u_i, v_i \leq n, u_i \neq v_i\)），表示树的一条边。

保证给定的边构成一棵树。保证所有测试用例中 \(n\) 的总和不超过 \(2×10^5\) 。

### 输出
对于每个测试用例，在一行中输出一个整数，表示答案。

### 示例
- **标准输入**
```
3
2
1 2
1 2
4
1 1 222 222
1 2
1 3
2 4
6
1 5 4 6 1 1
6 1
4 5
4 2
1 4
1 3
```
- **标准输出**
```
2
444
15
```

### 说明
在示例的第一个测试用例中，`dfs_order` 是唯一的，即 \([1, 2]\)，其值为 \(a_2 = 2\) 。注意 `dfs_order` 是基于1索引的。

在示例的第二个测试用例中，值最大的 `dfs_order` 是 \([1, 3, 2, 4]\)，其值为 \(a_3 + a_4 = 444\) 。
