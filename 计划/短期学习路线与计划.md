## 三、各阶段安排

### Phase 1：计算机网络（从零学）+ Docker + Git

> 预计 8-9 天（计网是全新的，给足时间）

**计网学习路线（主体）：**

| 顺序 | 内容 | 实验 |
|------|------|------|
| 1 | OSI/TCP/IP 体系结构、封包解包过程 | Wireshark 抓包体验 |
| 2 | 网络层：IP 协议、IP 分类、子网划分、ARP、ICMP | Wireshark 抓 ARP/ICMP 包 |
| 3 | **传输层：TCP 首部、三次握手、四次挥手**（重点） | Wireshark 抓 TCP 握手/挥手包，标注 SEQ/ACK |
| 4 | TCP 可靠传输：超时重传、快速重传、滑动窗口、拥塞控制 | `iptables` 模拟丢包观察现象 |
| 5 | UDP 协议、DNS 解析过程 | nslookup/dig 命令 + Wireshark 抓 DNS 包 |
| 6 | HTTP 协议：请求/响应格式、状态码、HTTP 版本演进 | `curl -v` 观察 HTTP 全过程 |
| 7 | HTTPS：SSL/TLS 握手过程、证书体系 | OpenSSL 查看证书链 |
| 8 | Cookie/Session、缓存机制、输入 URL 到页面展示全过程 | 用 C++ 手动构造 HTTP 请求并解析响应 |

**穿插完成（利用零碎时间）：**

| 内容 | 时间 |
|------|------|
| Docker 笔记：镜像/容器/Dockerfile/常用命令 | 半天 |
| Git 笔记：rebase/stash/冲突解决/分支策略 | 半天 |

**产出物：**
```
notes/network/01~08  +  抓包截图
Docker + Git 笔记
cpp/practice/http-client/  ← HTTP 客户端代码
```

### Phase 2：MySQL 进阶 + OS/Linux 快速过

> 预计 5 天（MySQL 是重点，OS/Linux 快速过）

**MySQL 进阶（3 天）：**

| 顺序 | 内容 | 实验 |
|------|------|------|
| 1 | 在你的 MySQL 笔记基础上扩充**索引篇**：B+ 树、聚簇/非聚簇索引 | 建 100w 数据表，EXPLAIN 对比加索引前后 |
| 2 | **事务 + MVCC**：隔离级别、MVCC 原理、undo log + ReadView | 开两个终端验证四种隔离级别 + 查 INNODB_TRX |
| 3 | **锁 + SQL 优化**：行锁/间隙锁/死锁、慢查询调优 | 制造死锁 + 抓慢 SQL 优化前后对比 |

**OS/Linux 快速过（2 天，你已有基础）：**

| 内容 | 方式 |
|------|------|
| 在你现有 OS 笔记基础上，补充 IPC 和死锁部分 | 写管道通信 + 死锁复现代码 |
| 更新 linux 指令笔记 | 补全原来缺的内容 |
| Shell 脚本 | 写 2 个常用脚本 |

**产出物：**
```
notes/mysql/    扩充完成（索引/事务/MVCC/锁/优化）
notes/os/       扩充完成（IPC/死锁）
notes/linux/    整理完成 + Shell 脚本
```

### Phase 3：Modern C++ + Java 补完 + OS/Linux 收尾

> 预计 6-7 天

**Modern C++（3 天）：**

| 顺序 | 内容 | 实验 |
|------|------|------|
| 1 | 智能指针：shared_ptr/unique_ptr/weak_ptr、循环引用 | 手写简易 shared_ptr |

**Java 补完（3 天）：**

| 顺序 | 内容 | 实验 |

| 3 | NIO：Channel/Buffer/Selector | NIO 写简易 HTTP 服务器 |
