# **【算法题】E. The 67th XOR Problem（第67个异或问题）**
每测试用例时间限制：3 秒
每测试用例内存限制：256 兆字节

### 题目描述
泛哺乳动物信息学奥林匹克竞赛（PMOI）其实也没那么糟糕——有传言说题目还不错，甚至（说出来可能遭天谴）很有趣！然而，令猕猴沮丧的是，三个不守规矩的参赛者（Cloud、ChatGBT和Grook）利用他们对整数序列在线百科（OEIS）的完美记忆开始作弊。这些无赖！突然之间，猕猴必须迅速行动，设计出一道这三人无法轻易作弊的题目。你，从他的伙伴被贬为他可怜的奴隶，被征召来进行测试。

给定一个数组 \(a\)，最初包含 \(n\) 个非负整数。

你恰好执行以下操作 \(n - 1\) 次：
1. 选择数组 \(a\) 的一个索引 \(i\)（\(1 \leq i \leq |a|\)，其中 \(|a|\) 表示数组当前的长度）。令 \(x = a_i\)。
2. 对于所有 \(1 \leq j \leq |a|\)，设置 \(a_j = a_j \oplus x\)，其中 \(\oplus\) 表示按位异或操作。
3. 从数组中移除 \(a_i\)。

可以证明，经过 \(n - 1\) 次操作后，数组中恰好剩下一个元素。你的任务是确定如果以最优方式执行这些操作，这个最终剩余元素的最大可能值。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 \(t\)（\(1 \leq t \leq 100\)）。接下来是测试用例的描述。

每个测试用例的第一行包含一个整数 \(n\)（\(2 \leq n \leq 3105\)）—— 数组的初始长度。

每个测试用例的第二行包含 \(n\) 个整数 \(a_1, a_2, \cdots, a_n\)（\(0 \leq a_i \leq 10^9\)）—— 数组的元素。

保证所有测试用例的 \(n\) 之和不超过 \(3105\) 。

### 输出
对于每个测试用例，输出一个整数 —— 最终元素的最大可能值。

### 示例
- **输入**
```
3
2
67 67
3
1 2 3
10
67 667 167 867 267 467 367 567 767 967
```
- **输出**
```
0
3
1012
```

### 说明
在第二个测试用例中，数组为 \([1, 2, 3]\) 。一种最优的操作序列如下：
1. 选择元素 \(3\) 。移除它。剩余元素变为 \([1 \oplus 3, 2 \oplus 3] = [2, 1]\) 。
2. 选择元素 \(2\) 。移除它。剩余元素变为 \([1 \oplus 2] = [3]\) 。
最终值为 \(3\) 。
