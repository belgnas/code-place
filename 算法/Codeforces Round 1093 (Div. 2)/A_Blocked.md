# **【算法题】Blocked（受阻）**
每个测试用例的时间限制：1 秒
每个测试用例的内存限制：256 兆字节

给定一个大小为\(n\)的整数数组\(a\)，如果\(a_i\)可以表示为\(a_1, a_2, \ldots, a_{i - 1}\)的一个子集的和（即存在\(1 \leq j_1 < j_2 < \ldots < j_k \leq i - 1\)，使得\(a_{j_1} + a_{j_2} + \ldots + a_{j_k} = a_i\)），我们就称位置\(1 \leq i \leq n\)是受阻的。重新排列数组\(a\)，使得没有位置受阻，或者报告这是不可能的。

例如，将数组\([3, 2, 5]\)重新排列为\([2, 3, 5]\)会使位置\(3\)受阻，因为我们可以表示\(5 = 2 + 3\)，但如果我们将其重新排列为\([3, 5, 2]\)，则没有位置受阻。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量\(t\)（\(1 \leq t \leq 400\)）。接下来是测试用例的描述。

每个测试用例的第一行包含一个整数\(n\)（\(1 \leq n \leq 200\)）。

第二行包含\(n\)个整数，表示数组\(a\)（\(1 \leq a_i \leq 100\)）。

### 输出
对于每个测试用例，如果存在一种排列使得没有位置受阻，则输出数组\(a\)的任意一种这样的排列，否则输出\(-1\)。

### 示例
**输入**
```
4
3
1 5 9
4
1 3 3 2
3
1 2 3
1
1
```
**输出**
```
5 9 1
-1
3 1 2
1
```

### 说明
在第三个测试用例中，数组\([3, 1, 2]\)没有受阻的位置：
- 位置\(1\)不受阻，因为\(3\)不能表示为\([]\)的一个子集的和。
- 位置\(2\)不受阻，因为\(1\)不能表示为\([3]\)的一个子集的和。
- 位置\(3\)不受阻，因为\(2\)不能表示为\([3, 1]\)的一个子集的和。
