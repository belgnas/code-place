# **【算法题】F. Best Player（最佳选手）**
每测试用例时间限制：3 秒
每测试用例内存限制：512 兆字节

### 题目描述
第17届烹饪格斗职业大赛（CCPC）已经结束，赛事主办方将选出本次比赛的最佳选手。

共有 \(n\) 名选手参赛，编号从 \(1\) 到 \(n\) ，他们总共进行了 \(m\) 场1对1的对决：
 - 第 \(i\) 场对决的双方是选手 \(a_i\) 和 \(b_i\)；
 - 每场对决分为上半场和下半场：
    - 在对决的上半场，选手 \(a_i\) 的得分是 \(x_i\) ，选手 \(b_i\) 的得分是 \(y_i\)；
    - 在对决的下半场，选手 \(a_i\) 和 \(b_i\) 的具体得分目前不确定，但两人得分之和是 \(z_i\)；
 - 一名选手在一场对决中的得分等于上半场得分与下半场得分之和。换句话说，在第 \(i\) 场对决中，选手 \(a_i\) 和 \(b_i\) 的可能得分分别是 \(x_i + p_i\) 和 \(y_i + q_i\) ，其中 \(0 \leq p_i, q_i \leq z_i\) 且 \(p_i + q_i = z_i\) 。

注意，所有得分均为非负整数，且每位选手至少参加了一场对决。

一名选手的最终得分是：他在参加的所有对决中取得的最高分。当且仅当一名选手的最终得分严格高于其他任何选手的最终得分时，他将赢得最佳选手奖。

由于 \(m\) 场对决下半场得分的不确定性，最佳选手也可能不同。请找出所有有可能成为最佳选手的选手。

### 输入
输入包含多个测试用例。
第一行包含一个整数 \(t\)（\(1 \leq t \leq 10^5\)），表示测试用例的数量。
对于每个测试用例：
 - 第一行包含两个整数 \(n\)，\(m\)（\(2 \leq n \leq 2×10^5\)，\(1 \leq m \leq 2×10^5\)），分别表示选手数量和对决场数。
 - 接下来 \(m\) 行，第 \(i\) 行包含5个整数 \(a_i\)，\(b_i\)，\(x_i\)，\(y_i\)，\(z_i\)（\(1 \leq a_i, b_i \leq n\)，\(a_i \neq b_i\)，\(0 \leq x_i, y_i, z_i \leq 10^5\)），其含义如上述描述。保证每位选手至少参加一场对决。

保证所有测试用例中 \(n\) 的总和与 \(m\) 的总和均不超过 \(2×10^5\) 。

### 输出
对于每个测试用例：
 - 第一行输出一个整数 \(k\) ，表示有可能成为最佳选手的选手数量。
 - 第二行按升序输出 \(k\) 个整数，表示有可能成为最佳选手的选手编号。特别地，当 \(k = 0\) 时，输出空行或不输出均视为正确。

### 示例
- **输入**
```
2
3 2
1 2 2 3 6
2 3 6 6 2
4 4
1 2 2 4 1
2 3 2 3 0
3 4 4 1 2
1 4 1 1 1
```
- **输出**
```
3
1 2 3
2
2 3
```

### 说明
在示例的第一个测试用例中，三名选手都有可能成为最佳选手。选手1只有在以下情况下可以成为最佳选手：
 - 在第一场对决中，选手1得分为 \(2 + 6 = 8\) ，选手2得分为 \(3 + 0 = 3\)；
 - 在第二场对决中，选手2得分为 \(6 + 1 = 7\) ，选手3得分为 \(6 + 1 = 7\) 。

在这种情况下，选手1的最终得分是 \(8\) ，选手2的最终得分是 \(\max(3, 7) = 7\) ，选手3的最终得分是 \(7\) ，所以选手1是最佳选手。
