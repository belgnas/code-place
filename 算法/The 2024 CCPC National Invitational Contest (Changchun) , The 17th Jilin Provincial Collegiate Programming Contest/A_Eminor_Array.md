# **【算法题】A. Eminor Array（Eminor数组）**
每测试用例时间限制：2 秒
每测试用例内存限制：512 兆字节

### 题目描述
格夫（Gew）正在寻找具有以下属性的“Eminor”序列 \([a_1, a_2, \cdots, a_m]\) ：
 - 序列非空（\(m \geq 1\)）；
 - \(1 \leq a_i \leq 2^n - 1\)；
 - 数组严格递增（对于每个 \(i \leq m - 1\)，有 \(a_i < a_{i + 1}\)）；
 - 不存在三个连续元素，其按位异或等于零（对于每个 \(i \leq m - 2\)，有 \(a_i \oplus a_{i + 1} \neq a_{i + 2}\) 。这里 \(\oplus\) 表示按位异或操作）。

现在，格夫好奇有多少个“Eminor”序列。由于“Eminor”序列的数量可能很大，你只需输出答案对 \(998244353\) 取模的结果。

### 输入
输入包含一个整数 \(n\)（\(1 \leq n \leq 10^6\)）。

### 输出
输出一个整数，表示“Eminor”序列的数量，对 \(998244353\) 取模的结果。

### 示例
- **输入**
```
1
```
- **输出**
```
1
```
- **输入**
```
2
```
- **输出**
```
6
```
- **输入**
```
3
```
- **输出**
```
91
```

### 说明
对于第二个测试用例，以下是 6 个可能的“Eminor”序列：
 - \([1]\)
 - \([2]\)
 - \([3]\)
 - \([1, 2]\)
 - \([1, 3]\)
 - \([2, 3]\)
