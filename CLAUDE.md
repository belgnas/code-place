# CLAUDE.md — 代码学习仓库

## 📋 项目概述

这是一个**个人代码学习与算法竞赛仓库**，用于记录、整理和回顾编程学习过程中的代码、笔记和算法题解。

- **语言**: C / C++ / Java
- **领域**: 语法学习、算法竞赛（CF / 牛客 / AtCoder / 蓝桥杯等）、操作系统理论
- **工具**: VS Code / VS（C++）、.cph（Competitive Programming Helper）插件
- **成就**: CCPC 东北区域赛铜牌

## 📝 文件命名规范

### 源码文件
- **C/C++**: 按知识点命名，如 `变量与scanf输入.c`、`高精度加法.cpp`
- **Java**: 按知识点命名，如 `变量与常量.java`、`OOP.java`；day 系列用 `day01-知识点/` 格式
- **算法题解 MD**: 使用题目标题英文或中文命名，位于对应比赛目录下

### 目录命名
- 根目录优先使用中文（专有名词除外）
- 学习路线用 `01-` 编号前缀，独立目录（如 `算法模板/` `练习/`）不带编号

## 🧠 算法目录说明

算法题解目录遵循 `比赛名/题目名.md` 的结构。每个 `.md` 通常对应同目录下同名的 `.cpp` 源码文件（cph 插件配套使用）。

支持的比赛平台：
- **Codeforces** (Div. 2 / Div. 3 / Div. 4 / Edu Round)
- **牛客周赛 / 牛客练习赛 / 牛客挑战赛**
- **AtCoder Beginner Contest**
- **蓝桥杯多校模拟赛**
- **CCPC / 区域赛 / 省赛**
- **Whalica Cup**

## ⚙️ 开发环境

- **C++**: VS（Visual Studio）+ .cph 插件
- **算法**: VS Code + .cph 插件（自动关联 cpp 和对应的 md 题解）
- **Git**: 常规 git 工作流，提交信息以日期和内容描述为主

## 🤝 对 Claude 的要求

1. **定位准确** — 查找文件时，优先通过目录结构和文件命名规律推断
2. **中文优先** — 除非涉及源码标识符，否则使用中文交流
3. **保持风格** — 写代码/笔记时保持与库内现有文件一致的命名和注释风格
4. **算法辅助** — 遇到算法题时，提供思路推导、复杂度分析和完整代码
5. **避免冗余** — 不要向 `.cph/` `output/` `__pycache__/` 等构建/缓存目录写文件
