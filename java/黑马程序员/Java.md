# Java相关

## Java内存分配
1. 栈区
   * 方法运行时使用的内存，比如main方法运行，进入方法栈中执行
   * 多个方法入栈时，按照先进后出的顺序出栈
2. 堆区
   * 存储对象或者数组，new来创建的，都存储在堆内存
3. 方法区
   * 存储可以运行的class文件
4. 本地方法栈
   * JVM在使用操作系统功能的时候使用，和我们开发无关
5. 寄存器
   * 给CPU使用，和我们开发无关

## Java方法
1. 什么是方法?
   * 方法是程序中最小的执行单元，
2. 实际开发中，什么时候用到方法?
   * 重复的代码、具有独立功能的代码可以抽取到方法中，
3. 实际开发中，方法有什么好处?
   * 可以提高代码的复用性
   * 可以提高代码的可维护性
4. 方法调用的基本内存原理
   * 方法调用时，会在栈区开辟一个新的空间，称为栈帧
   * 方法调用结束后，会在栈区释放这个空间
5. 方法传递基本数据类型的内存原理
   * 在栈帧中开辟一个新的空间，存储数据值
6. 方法传递引用数据类型的内存原理
   * 引用数据类型中存储的是地址值，指向堆内存中的对象

## 面向对象编程（OOP）
1. 什么是面向对象编程？
   * 面向对象编程是一种编程思想，是对客观世界的一种理解和描述的方法。
   * 面向对象编程的基本特征是：封装、继承、多态。
2. 什么是类？
   * 类是对现实生活中一类具有共同特征的事物的抽象描述。
   * 类是对象的数据类型，类是具有相同属性和行为的一组对象的集合。
3. 什么是对象？
   * 对象是类的一个实例，是具体的事物。
   * 对象是内存中的一块区域，用于存储对象的属性和方法。
   * 对象是通过new关键字创建的。
4. 建议一个java文件只有一个类，类名与文件名要保持一致。
5. 封装
   * 封装是指将类的属性和方法进行有机结合，隐藏对象的属性和实现细节，仅对外提供公共的访问方式。
   * 封装的好处是：提高了代码的复用性，提高了代码的可维护性，隐藏了代码的实现细节，提高了代码的安全性。
6. API
   * API（Application Programming Interface）应用程序编程接口，是一些列的程序设计语言的函数和数据结构，用于实现应用程序的功能。
   * API的作用是：提供一些列的方法，供其他程序使用。
   * API的分类：
     * 内置API：Java语言提供的类和方法，直接使用即可。
     * 第三方API：其他公司提供的类和方法，需要导入jar包才能使用。

7. 字符串String
   * ps：String类的对象是不可变的，一旦被创建，就不能被改变。
   * 字符串存储的内存原理
     * 直接赋值会复用字符串常量池中的new出来不会复用，而是开辟一个新的空间
   * 比较：
     * 对于==来说
       * 基本数据类型，比较的是数据值。
       * 引用数据类型，比较的是地址值。
     * 对于equals方法来说
       * 比较的是字符串的内容是否相同
       * 比较的是字符串的地址值是否相同
   * 字符串拼接的底层原理
     * 如果没有变量参与，都是字符串直接相加，编译之后就是拼接之后的结果，会复用串池中的字符串
     * 如果有变量参与，每一行拼接的代码，都会在内存中创建新的字符串，浪费内存。
   * StringBuilder提高效率原理图
     * 所有要拼接的内容都会往StringBuilder中放，不会创建很多无用的空间，节约内存
   * StringBuilder源码分析
     * 默认创建一个长度为16的字节数组添加的内容长度小于16，直接存添加的内容大于16会扩容(原来的容量*2+2)，如果扩容之后还不够，以实际长度为准。
