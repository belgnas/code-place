# **【算法题】I Wanna Make It Palindromic（我想让它成为回文序列）**

### 比赛主页
 - 我的提交

- **时间限制**：C/C++/Rust/Pascal 2 秒，其他语言 4 秒
- **空间限制**：C/C++/Rust/Pascal 256 M，其他语言 512 M
- **64bit IO 格式**：%lld

### 题目描述 
给定一个仅由绝对值不超过 $200$ 的整数构成的序列 $s_1, s_2, \ldots, s_n$ 。你需要先选定两个整数 $p_1, p_2$（可相等，选定后在本组内不可修改），随后可以执行任意多次（可为 $0$ 次）如下操作：选择一个位置 $i$，将 $s_i$ 改为 $p_1$ 或 $p_2$，消耗 $1$ 枚金币。

求最少的消耗金币数，使得序列为回文序列。

【名词解释】
 - **回文序列**：一个序列被称作回文序列，当且仅当这个序列从左往右读和从右往左读是相同的。

### 输入描述:
每个测试文件均包含多组测试数据。第一行输入一个整数 $T(1 \leq T \leq 100)$ 代表数据组数，每组测试数据描述如下：
 - 第一行输入一个整数 $n(1 \leq n \leq 2×10^5)$，表示序列的长度；
 - 第二行输入 $n$ 个整数 $s_1, s_2, \ldots, s_n(-200 \leq s_i \leq 200)$，表示序列中的元素。

除此之外，保证单个测试文件的 $n$ 之和不超过 $2×10^5$ 。

### 输出描述:
对于每一组测试数据，新起一行输出一个整数，表示最少的消耗金币数。

### 示例1

**输出**
```

4
```
