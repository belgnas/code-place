# **【算法题】C. Specialty String（特殊字符串）**
每测试用例时间限制：2 秒
每测试用例内存限制：256 兆字节

### 题目描述
AksLolCoding正在对一个长度为 \(n\) 的字符串 \(s\) 玩游戏。最初，\(s\) 仅包含小写拉丁字母。

在一轮中，AksLolCoding 可以选择任意一对整数 \((i, j)\) ，需满足：
 - \(1 \leq i < j \leq n\)；
 - \(s_i = s_j \neq *\)；并且
 - 对于所有 \(i < k < j\) ，都有 \(s_k = *\) 。

如果不存在这样的 \(i, j\) ，则游戏结束。否则，AksLolCoding 将 \(s_i\) 设为 \(*\) ，并将 \(s_j\) 设为 \(*\) 。

一旦游戏结束，当且仅当 \(s\) 中的每个字符都等于 \(*\) 时，AksLolCoding 获胜。判断 AksLolCoding 是否有可能获胜。

### 说明
\(*\) 代表 ASCII 码为 42 的字符。

### 输入
第一行包含一个整数 \(t\)（\(1 \leq t \leq 100\)），表示测试用例的数量。

每个测试用例的第一行包含一个整数 \(n\)（\(1 \leq n \leq 5000\)），即字符串的长度。

每个测试用例的第二行包含一个由小写拉丁字母组成的字符串 \(s\) 。

所有测试用例的 \(n\) 之和不超过 \(5000\) 。

### 输出
在单独的行上输出每个测试用例的答案。如果 AksLolCoding 能获胜，输出 “YES”（不含引号）。否则，输出 “NO”（不含引号）。

你可以以任何大小写形式输出答案。例如，字符串 “yEs”、“yes”、“Yes” 和 “YES” 都将被识别为肯定回答。

### 示例
- **输入**
```
6
1
a
6
llmllm
6
uwuuwu
6
byebye
6
oooioi
12
siixxsevvenn
```
- **输出**
```
NO
YES
YES
NO
NO
YES
```

### 说明
在第一个测试用例中，可以证明 AksLolCoding 不可能获胜。

在第二个测试用例中，AksLolCoding 可以通过以下步骤获胜：llmllm → llm\*\*m → \*\*m\*\*m → \*\*\*\*\*\* 。

在第三个测试用例中，AksLolCoding 可以通过以下步骤获胜：uwuuwu → uw\*\*wu → u\*\*\*\*u → \*\*\*\*\*\* 。
