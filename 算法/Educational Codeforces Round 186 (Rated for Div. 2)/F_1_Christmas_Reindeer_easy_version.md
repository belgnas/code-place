# **【算法题】F1. Christmas Reindeer (easy version)（圣诞驯鹿（简单版））**
每测试用例时间限制：4 秒
每测试用例内存限制：512 兆字节

### 题目描述
这是该问题的简单版本。两个版本的唯一区别在于 \(n\) 和 \(m\) 的上限。在这个版本中，\(n \leq 500\) 且 \(m \leq 500\) 。

你有一群 \(n\) 只圣诞驯鹿。第 \(i\) 只驯鹿的力量值为 \(2^{c_i}\) 。

一群 \(k\) 只圣诞驯鹿的承载能力计算方式如下：
 - 将驯鹿的力量值按非递增顺序排序。设排序后的力量值列表为 \(c'_1, c'_2, \cdots, c'_k\) ，其中 \(c'_i \geq c'_{i + 1}\) ；
 - 那么，这群驯鹿的承载能力等于 \(c'_1 + \lfloor \frac{c'_2}{2} \rfloor + \lfloor \frac{c'_3}{4} \rfloor + \cdots + \lfloor \frac{c'_k}{2^{k - 1}} \rfloor\) 。

注意，有些驯鹿可能对群体的承载能力贡献为零。

你需要处理三种类型的查询：
 - 向鹿群中添加一只力量值为 \(2^x\) 的驯鹿；
 - 从鹿群中移除一只力量值为 \(2^x\) 的驯鹿；
 - 计算从鹿群中选择一些驯鹿（可能是全部），使得所选群体的承载能力至少为 \(x\) 的方法数。

如果鹿群中有多只力量值相同的驯鹿，它们被视为不同的个体。例如，如果你有两只力量值都为 \(1\) 的驯鹿，并且你需要计算选择承载能力至少为 \(1\) 的群体的方法数，有 \(3\) 种选择方式：选择第一只驯鹿、选择第二只驯鹿，或者两只都选。

### 输入
第一行包含两个整数 \(n\) 和 \(m\)（\(1 \leq n, m \leq 500\)）—— 鹿群中初始驯鹿的数量和查询的数量，分别表示。

第二行包含 \(n\) 个整数 \(c_1, c_2, \cdots, c_n\)（\(0 \leq c_i \leq 60\)），表示鹿群中驯鹿的力量值：第 \(i\) 只驯鹿的力量值为 \(2^{c_i}\) 。

接下来的 \(m\) 行以以下格式之一描述查询：
 - \(1\) \(x\)（\(0 \leq x \leq 60\)）—— 向鹿群中添加一只力量值为 \(2^x\) 的驯鹿；
 - \(2\) \(x\)（\(0 \leq x \leq 60\)）—— 从鹿群中移除一只力量值为 \(2^x\) 的驯鹿；
 - \(3\) \(x\)（\(1 \leq x \leq 10^{18}\)）—— 计算从鹿群中选择一群驯鹿，使得所选群体的承载能力至少为 \(x\) 的方法数。

输入的附加限制：每当给出类型为 \(2\) 的查询时，当前鹿群中至少有一只力量值为 \(2^x\) 的驯鹿。

### 输出
对于每个类型为 \(3\) 的查询，输出一个整数 —— 从鹿群中（可能选择整个鹿群）选择一群驯鹿，使得所选群体的承载能力至少为 \(x\) 的方法数。由于结果可能很大，输出对 \(998244353\) 取模的结果。

### 说明
让我们考虑第一个示例。最初，有三只驯鹿，力量值分别为 \(4\)、\(2\) 和 \(2\) 。
 - 在第一个查询中，你需要计算选择承载能力至少为 \(5\) 的群体的方法数。有三种可能的群体：\(\{1, 2\}\)（包含第 \(1\) 只和第 \(2\) 只驯鹿的群体）、\(\{1, 2, 3\}\) 和 \(\{1, 3\}\) ；
 - 在第二个查询中，你需要计算选择承载能力至少为 \(6\) 的群体的方法数。即使整个鹿群的承载能力也只有 \(4 + \lfloor \frac{2}{2} \rfloor + \lfloor \frac{2}{4} \rfloor = 5\) ，所以没有合适的群体选择方式；
 - 在第三个查询中，添加了一只力量值为 \(4\) 的驯鹿。我们将其记为第 \(4\) 只驯鹿；
 - 在第四个查询中，可能的群体是 \(\{1, 4\}\)、\(\{1, 2, 4\}\)、\(\{1, 3, 4\}\) 和 \(\{1, 2, 3, 4\}\) ；
 - 在第五个查询中，有 \(10\) 种可能的群体；
 - 在第六个查询中，移除了一只力量值为 \(2\) 的驯鹿。假设它是第 \(2\) 只驯鹿，所以只剩下驯鹿 \(1\)、\(3\)、\(4\) ；
 - 在第七个查询中，你需要计算选择承载能力至少为 \(5\) 的群体的方法数。有四种可能的群体：\(\{1, 3\}\)、\(\{1, 4\}\)、\(\{1, 3, 4\}\)、\(\{3, 4\}\) 。
