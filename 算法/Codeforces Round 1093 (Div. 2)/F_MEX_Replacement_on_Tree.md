# **【算法题】F. MEX Replacement on Tree（树上的MEX替换）**
每个测试用例的时间限制：3.5 秒
每个测试用例的内存限制：256 兆字节

给定一棵以 \(1\) 为根节点的树，它有 \(n\) 个顶点，以及一个长度为 \(n\) 的排列 \(p\)，该排列由整数 \(0, 1, \ldots, n - 1\) 组成，其中 \(p_v\) 表示顶点 \(v\) 的权重。

设 \(S_v\) 表示从根节点 \(1\) 到顶点 \(v\) 的路径上各顶点权重组成的集合。定义函数 \(f\) 为 \(f(v) = MEX(S_v)\)，即根节点到 \(v\) 的简单路径（包括两端点）上各顶点权重组成集合的 \(MEX\) 值，其中整数集合 \(c_1, c_2, \ldots, c_k\) 的 \(MEX\) 定义为未在集合 \(c\) 中出现的最小非负整数 \(x\)。

同时，给定以下操作：

选择一个整数 \(v\)（\(1 \leq v \leq n\)），并将 \(f(v)\) 赋值给 \(p_v\)。

你的目标是在执行上述操作不超过一次（也可能不执行）的情况下，找到 \(\sum_{v = 1}^{n} f(v)\) 的最大值。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 \(t\)（\(1 \leq t \leq 2 \cdot 10^4\)）。接下来是测试用例的描述。

每个测试用例的第一行包含一个整数 \(n\)（\(1 \leq n \leq 2 \cdot 10^5\)）——树的顶点数量。

每个测试用例的第二行包含 \(n\) 个整数 \(p_1, p_2, \ldots, p_n\)（\(0 \leq p_i \leq n - 1\)）——数组 \(p\) 的元素。保证从 \(0\) 到 \(n - 1\) 的每个整数在 \(p\) 中恰好出现一次。

接下来 \(n - 1\) 行，每行包含两个整数 \(x_i\) 和 \(y_i\)（\(1 \leq x_i, y_i \leq n\)），表示顶点 \(x_i\) 和 \(y_i\) 之间有一条边。保证这些边构成一棵树。

保证所有测试用例中 \(n\) 的总和不超过 \(2 \cdot 10^5\)。

### 输出
对于每个测试用例，输出一行包含一个整数：在执行上述操作不超过一次（也可能不执行）的情况下，\(\sum_{v = 1}^{n} f(v)\) 的最大值。

### 说明
- 在第一个示例中，不进行任何操作是最优的，所以答案是 \(1\)。
- 在第二个示例中，我们有：
    - \(f(1) = MEX(\{p_1\}) = MEX(\{1\}) = 0\)
    - \(f(2) = MEX(\{p_1, p_2\}) = MEX(\{1, 0\}) = 2\)
    - \(f(3) = MEX(\{p_1, p_3\}) = MEX(\{1, 2\}) = 0\)

我们有以下几种选择：
    - 不进行操作，\(\sum_{v = 1}^{n} f(v) = 2\)。
    - 对顶点 \(1\) 进行操作，将 \(f(1) = 0\) 赋值给 \(p_1\)，\(f(1), f(2), f(3)\) 变为 \(1\)，此时 \(\sum_{v = 1}^{n} f(v) = 3\)。
    - 对顶点 \(2\) 进行操作，将 \(f(2) = 2\) 赋值给 \(p_2\)，\(f(2)\) 变为 \(0\)，此时 \(\sum_{v = 1}^{n} f(v) = 0\)。
    - 对顶点 \(3\) 进行操作，将 \(f(3) = 0\) 赋值给 \(p_3\)，\(f(3)\) 变为 \(2\)，此时 \(\sum_{v = 1}^{n} f(v) = 4\)。

所以答案是 \(4\)，通过对顶点 \(3\) 进行操作得到。
