# **【算法题】D. Antiamuny Wants to Learn Swap（安蒂亚穆尼想学交换操作）**
- **每个测试的时间限制**：2 秒
- **每个测试的内存限制**：512 兆字节

对于长度为 $m$ 的数组 $b$，你可以执行以下两种操作：
1. 选择一个索引 $1 \leq i \leq m - 1$，然后交换 $b_i$ 和 $b_{i + 1}$ 的值。
2. 选择一个索引 $1 \leq i \leq m - 2$，然后交换 $b_i$ 和 $b_{i + 2}$ 的值。

然而，操作 2 最多只能执行一次。

我们定义 $f(b)$ 为将数组 $b$ 按非递减顺序排序所需的（同时使用操作 1 和操作 2）最少操作次数，$g(b)$ 为仅使用操作 1 将数组 $b$ 按非递减顺序排序所需的最少操作次数。

如果 $f(b) = g(b)$，则数组 $b$ 是完美的。换句话说，与仅使用相邻交换操作相比，能够使用操作 2 并不会减少将数组 $b$ 排序所需的操作次数。

给定一个长度为 $n$ 的排列 $a$∗，并且必须回答 $q$ 个查询。每个查询由两个整数 $l$ 和 $r$（$1 \leq l \leq r \leq n$）组成，表示子数组 $a[l \ldots r]$†。对于每个查询，判断子数组 $a[l \ldots r]$ 是否完美。

∗ 长度为 $n$ 的排列是一个由从 $1$ 到 $n$ 的 $n$ 个不同整数按任意顺序组成的数组。例如，$[2, 3, 1, 5, 4]$ 是一个排列，但 $[1, 2, 2]$ 不是排列（数组中 2 出现了两次），$[1, 3, 4]$ 也不是排列（$n = 3$ 但数组中有 4）。

† 子数组 $a[l \ldots r]$ 包括从索引 $l$ 到 $r$ 的所有元素，即 $[a_l, a_{l + 1}, a_{l + 2}, \ldots, a_r]$。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 $t$（$1 \leq t \leq 5 \cdot 10^4$）。接下来是测试用例的描述。

每个测试用例的第一行包含两个整数 $n$，$q$（$1 \leq n, q \leq 5 \cdot 10^5$）—— 数组 $a$ 的长度和查询的数量。

每个测试用例的第二行包含 $n$ 个整数 $a_1, a_2, \ldots, a_n$（$1 \leq a_i \leq n$）—— 排列 $a$ 中的元素。

接下来的 $q$ 行，每行包含两个整数 $l$ 和 $r$（$1 \leq l \leq r \leq n$）—— 查询的子数组的左右端点。

保证所有测试用例中 $n$ 的总和以及 $q$ 的总和均不超过 $5 \cdot 10^5$。

### 输出
对于每个测试用例，如果查询的子数组 $a[l \ldots r]$ 是完美的，输出 “YES”，否则输出 “NO”。

你可以以任何大小写形式输出答案。例如，字符串 “yEs”、“yes”、“Yes” 和 “YES” 都将被识别为肯定回答。

### 示例
**输入**
```
2
5 5
1 5 4 3 2
1 2
1 5
3 5
1 4
2 5
5 5
3 2 1 4 5
1 1
4 5
1 4
2 5
3 4
```
**输出**
```
YES
NO
NO
NO
NO
YES
YES
NO
YES
YES
```

### 注意
在第一个测试用例中：
 - **查询 1**：$a[1 \ldots 2] = [1, 5]$ 已经按升序排序。因此，$f(a[1 \ldots 2]) = g(a[1 \ldots 2]) = 0$，子数组 $a[1 \ldots 2]$ 是完美的。
 - **查询 2**：$a[1 \ldots 5] = [1, 5, 4, 3, 2]$。$f(a[1 \ldots 5]) = 4$，因为我们可以使用以下操作序列对数组进行排序：
$[1, 5, 4, 3, 2] \xrightarrow{\text{op2}} [1, 3, 4, 5, 2] \xrightarrow{\text{op1}} [1, 3, 4, 2, 5] \xrightarrow{\text{op1}} [1, 3, 2, 4, 5] \xrightarrow{\text{op1}} [1, 2, 3, 4, 5]$

另一方面，$g(a[1 \ldots 5]) = 6$，因为至少需要 6 次相邻交换才能对 $a[1 \ldots 5]$ 进行排序。由于 $f(a[1 \ldots 5]) \neq g(a[1 \ldots 5])$，子数组 $a[1 \ldots 5]$ 不是完美的。
 - **查询 3**：$a[3 \ldots 5] = [4, 3, 2]$。$f(a[3 \ldots 5]) = 1$，因为我们可以使用以下操作序列对数组进行排序：
$[4, 3, 2] \xrightarrow{\text{op2}} [2, 3, 4]$

另一方面，$g(a[3 \ldots 5]) = 3$，因为至少需要 3 次相邻交换才能对 $a[3 \ldots 5]$ 进行排序。由于 $f(a[3 \ldots 5]) \neq g(a[3 \ldots 5])$，子数组 $a[3 \ldots 5]$ 不是完美的。
