## 四、C++ 后端 — 详细执行计划（推荐路线）

> 总时长预估：**6-8 个月**，目标：做到能投递大厂日常实习/秋招的水平

### 阶段一：Modern C++ 入门（2-3 周）

```
学习目标：
├── C++11/14 核心新特性
│   ├── auto / decltype / 范围 for
│   ├── nullptr / override / final
│   ├── lambda 表达式
│   ├── 智能指针（shared_ptr / unique_ptr / weak_ptr）
│   └── 右值引用 + 移动语义（move / forward）
├── 标准库补充
│   ├── unordered_map/set
│   ├── chrono 时间库
│   ├── thread / mutex / condition_variable
│   └── filesystem
│
参考书：《Effective Modern C++》前 4 章
练习：把仓库里旧的 C++ 语法练习用新特性重写几个
```

### 阶段二：Linux 系统编程（2-3 周）

```
学习目标：
├── Linux 基础
│   ├── 常用命令（你已经有一部分笔记了）
│   ├── GCC / GDB / Makefile / CMake
│   └── 静态库和动态库
├── 系统编程
│   ├── 文件 I/O（open/read/write/lseek）
│   ├── 进程（fork/exec/wait）
│   ├── 线程（pthread）
│   ├── IPC（管道/消息队列/共享内存/信号量）
│   └── 信号处理
│
参考书：《APUE》前 10 章（选读）
练习：写一个简单的文件传输程序
```

### 阶段三：网络编程（3-4 周）

```
学习目标：
├── TCP/IP 协议基础
│   ├── TCP 三次握手/四次挥手
│   ├── TCP 流量控制/拥塞控制
│   └── IP / 路由基础
├── Socket 编程
│   ├── socket / bind / listen / accept / connect
│   ├── 多进程/多线程并发服务器
│   └── I/O 多路复用（select / poll / epoll）
├── Reactor 模型
│   └── 事件驱动 + 非阻塞 I/O
│
参考书：《Unix 网络编程》第 1-6 章 + 第 16 章
参考项目：muduo 网络库（只看核心设计思路）
```

### 阶段四：核心项目（6-8 周）⭐ 最关键的一步

> **必做项目：C++ 高并发 WebServer**

```
项目要求：
├── 使用 C++11/14
├── 基于 epoll 的 Reactor 事件驱动模型
├── 线程池（避免频繁创建线程）
├── 解析 HTTP GET / POST 请求
├── 返回静态文件（HTML/CSS/JS）
├── 支持 Keep-Alive 长连接
├── 压力测试（使用 WebBench 或 wrk，QPS 达到一定指标）
│
推荐实现路线：
1. 先写一个单线程 accept 版本（1天）
2. 加多线程 + 线程池（2天）
3. 改成 epoll + Reactor（1周）
4. 解析 HTTP 协议（1周）
5. 优化 + 压测 + 写 README（1周）
```

**扩展加分项目**（时间充裕时选做）：
- 加上日志系统（同步/异步日志）
- 加上定时器（处理非活跃连接）
- 加上内存池（减少 malloc 次数）
- 配上配置文件解析

### 阶段五：数据库学习（2-3 周）

```
MySQL（你已经有了基础笔记，深入学）
├── 索引原理（B+ 树）

└── 给你的 WebServer 加上 MySQL 存储

Redis

└── cpp 客户端操作 Redis
```

## 六、我的核心建议

### 🎯 当前的优先级排序

```
1️⃣ 仓库重构 ─── 整理知识资产，建立信心
2️⃣ 掌握 Modern C++ ─── 从 C++98 升级到 C++17（2-3 周）
3️⃣ 完成 WebServer ─── 你的第一个拿得出手的项目（6-8 周）
4️⃣ 深入 MySQL + Redis ─── 后端必备技能
5️⃣ 系统刷八股文 + 面经 ─── 把算法优势转化为面试优势
```

### 💡 几个关键认知

1. **CCPC 铜牌已经证明了你的学习能力** — 这块牌子在大厂 C++ 岗简历筛选时是亮眼的加分项。不要觉得它"不够金"，对一个自学/非名校学生来说，它比很多课程项目都有说服力。

2. **现在的核心矛盾不是"学得慢"，而是"学到的没转化成产出"** — 你学了 1 年半没有做一个完整的项目，这才是真实的问题。WebServer 不是可选项，是必须项。

3. **算法竞赛基础让你在面试刷题阶段比普通人轻松 2-3 倍** — 别人刷 LeetCode 200 题才能掌握的技巧，你 CCPC 选手可能 50 题就够了。这个时间优势要利用好。

4. **保研不是唯一出路** — 错过保研不代表什么。算法竞赛拿牌 + 一个扎实的项目 + 好好准备，拿到的 offer 可能比保研同学更好。

### 📌 如果要选一个方向

> **建议优先走 C++ 后端路线**。理由有三：
> 1. 你的 C++ 和算法积累最强，转换成本最低
> 2. CCPC 铜牌在 C++ 岗面试中是最直接的优势证明
> 3. Java 赛道卷度更大，且你的 Java 基础还比较薄弱
