# **【算法题】[習]ウミユリ海底譚**

**时间限制**: C/C++/Rust/Pascal 2秒，其他语言4秒  
**空间限制**: C/C++/Rust/Pascal 1024 M，其他语言2048 M  
**Special Judge, 64bit IO Format**: %lld

## 题目描述

![图1：辅助示意图](https://uploadfiles.nowcoder.com/images/20260408/0_1775610715520/9EA37FF98E658E692DE3FBEC18C19AE9)

去年校赛，秋绘穿越到了一个舞萌的歌曲完成率取决于最大连击分数的平行宇宙，顿悟并鸟……啊不对，鸟加了《ウミユリ海底譚》。

一年过去了，秋绘已经从一个 w3 的小萌新变成了一个 w5 的高级萌新，开始琢磨着刷海底谭的理论值了。这不，舞萌更新了宴谱《[習]ウミユリ海底譚》，身为海底谭爱好者的秋绘当然也要品鉴一番，顺便看看能不能帮助她刷海底谭理论。

总而言之，秋绘想尽快达成海底谭理论值。为了量化这一过程，我们定义了两个关键属性：**熟练度**和**手癖度**。

已知熟练度的上限为 $x$，手癖度的上限为 $y$。假设秋绘当前的熟练度为 $a$ ($0 \le a \le x$)，当前的手癖度为 $b$ ($0 \le b \le y$)。

每一次操作（也就是打一局游戏），秋绘可以根据当前状态，在以下两种策略中选择一种：

1. **打《ウミユリ海底譚》（标准谱面）：**
   - 她有 $\frac{a}{x} \times (1 - \frac{b}{y})$ 的概率成功达成理论值，如果成功，则过程结束。
   - 如果未能达成理论值，她的熟练度会因为练习而提升，但手癖（错误的肌肉记忆）也会加深。此时，熟练度变为 $\min(x, a + k_1)$，手癖度变为 $\min(y, b + 1)$。

2. **打《[習]ウミユリ海底譚》（宴谱）：**
   - 这是一个专门为了方便玩家练习海底谭而设计的谱面。打这个谱面无法达成标准谱面的理论值，操作次数必然 $+1$。
   - 但是，她的熟练度会提升，且手癖度会降低。此时，熟练度变为 $\min(x, a + k_2)$，手癖度变为 $\max(0, b - 1)$。

给定初始熟练度 $a_0$，初始手癖度 $b_0$，以及参数 $x, y, k_1, k_2$。请你帮助秋绘计算，在采取最优策略的前提下，她期望需要进行多少次操作才能成功达成海底谭理论值。

## 输入描述

每个测试文件包含多组测试数据。第一行包含一个正整数 $T$，表示测试数据的组数。接下来是测试数据的描述。

每组测试数据仅包含一行六个正整数 $a_0, b_0, x, y, k_1, k_2$ （$1 \le a_0, k_1, k_2 \le x$，$1 \le b_0 \le y$，$1 \le x \times y \le 5 \times 10^6$）—— 分别表示初始熟练度、初始手癖度、熟练度上限、手癖度上限以及题目描述中的两个参数。

保证所有测试数据中 $x \times y$ 的和不超过 $10^7$。

## 输出描述

对于每组测试数据，输出一行一个浮点数，表示秋绘达成理论值期望需要的操作次数。

假设你的输出为 $a$，标准答案为 $b$，若满足以下任意一个条件，则认为你的答案正确：

$$|a - b| < 10^{-6} \quad \text{or} \quad \frac{|a - b|}{|b|} < 10^{-6}$$

其中 $|a - b|$ 定义为**绝对误差**，$\frac{|a - b|}{|b|}$ 定义为**相对误差**。

## 示例 1

**输入**

```
3
2 1 2 2 1 2
5 5 5 5 1 3
6 1 15 3 1 14
```

**输出**

```
2.000000000000
5.333333333333
2.000000000000
```

```

```

```
4.000000000000
2.000000000000
1.955687242798
```
