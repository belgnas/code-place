# **【算法题】B. Party Monster**

**时间限制**: 2秒  
**内存限制**: 256 MB

## 题目描述

尤瑟夫给了你一个长度为 $n$ 的序列 $s$，仅由字符 `(` 和 `)` 组成。你最多可以执行一次以下操作：

- 从 $s$ 中选择一个子串并移除它。然后，你可以将移除的字符**逐个重新插入**到剩下的字符串中。每个字符可以独立地放置在任意位置（可以与其他字符分开，插入任意位置）。

尤瑟夫希望你判断，是否可能在最多执行一次操作后，得到一个正则括号序列。

**定义**：
- **子串**：一个字符串的连续子段。例如，`"acab"` 是 `"abacaba"` 的一个子串（从位置 $3$ 开始到位置 $6$ 结束），但 `"aa"` 或 `"d"` 不是该字符串的子串。字符串 $s$ 从位置 $l$ 到 $r$ 的子串记为 $s[l,r] = s_l s_{l+1} \ldots s_r$。
- **正则括号序列**：是指可以在序列的字符之间插入字符 `1` 和 `+` 后转化为一个正确的算术表达式的括号序列。例如：
  - 括号序列 `()()` 和 `(())` 是正则的（得到的表达式分别为：`(1)+(1)` 和 `((1+1)+1)`）；
  - 括号序列 `)(`、`(` 和 `)` 不是正则的。

## 输入格式

第一行包含一个整数 $t$ ($1 \le t \le 10^4$) — 测试用例的数量。接下来是对各个测试用例的描述。

每个测试用例的第一行包含一个整数 $n$ ($1 \le n \le 2 \cdot 10^5$) — 字符串 $s$ 的长度。

每个测试用例的第二行包含一个长度为 $n$ 的序列 $s$，仅由字符 `(` 和 `)` 组成。

保证所有测试用例的 $n$ 之和不超过 $2 \cdot 10^5$。

## 输出格式

对于每个测试用例，如果可以将序列变为正则括号序列，则输出 `YES`，否则输出 `NO`。

你可以以任意大小写形式输出答案。例如，字符串 `yEs`、`yes`、`Yes` 和 `YES` 都将被识别为肯定答复。

## 示例

**输入**

```
6
2
()
2
)(
3
(((
6
())(()
4
(()(
5
)()()
```

**输出**

```
YES
YES
NO
YES
NO
NO
```

## 说明

在第一个测试用例中，字符串 $s$ 已经是一个正则括号序列，因此答案是 `YES`。

在第二个测试用例中，我们可以移除子串 $s[2,2] = \texttt{(}$，并将其重新插入到字符串开头，得到 $s = \texttt{()}$，因此答案是 `YES`。

在第三个测试用例中，无法通过操作得到一个正则括号序列，因此答案是 `NO`。

在第四个测试用例中，我们可以选择子串 $s[3,4]=\texttt{)(}$，将其移除，然后按如下方式重新插入字符：

$$ \texttt{())(()} \rightarrow \texttt{()()} \rightarrow \texttt{(()())} $$

因此我们得到了一个正则括号序列，答案为 `YES`。
