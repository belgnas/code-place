# **【算法题】A. Eating Game（进食游戏）**
每测试用例时间限制：1 秒
每测试用例内存限制：256 兆字节

### 题目描述
有 \(n\) 个玩家在一张圆桌上玩游戏。第 \(i\) 个玩家有 \(a_i\) 道菜要吃。他们轮流进食，任何一个玩家都可以先开始。

在轮到某个玩家时，如果玩家 \(i\) 还有剩余的菜，那么他必须恰好吃掉一道菜。然后，玩家 \((i \bmod n) + 1\) 开始他的回合。这个过程持续到所有的菜都被吃完。

吃掉最后一道菜的玩家被视为获胜者。确定可能成为获胜者的玩家数量。

### 输入
第一行包含一个整数 \(t\)（\(1 \leq t \leq 5000\)），表示测试用例的数量。

每个测试用例的第一行包含一个整数 \(n\)（\(1 \leq n \leq 10\)）。

每个测试用例的第二行包含 \(n\) 个整数，即数组 \(a\) 的元素（\(1 \leq a_i \leq 10\)）。

### 输出
对于每个测试用例，输出一行答案。

### 示例
- **输入**
```
3
1
10
2
6 7
4
1 4 3 4
```
- **输出**
```
1
1
2
```

### 说明
在第一个测试用例中，无论哪个玩家先开始，玩家 1 都会获胜。

在第二个测试用例中，无论哪个玩家先开始，玩家 2 都会获胜。

在第三个测试用例中，玩家 2 和玩家 4 可能获胜。
