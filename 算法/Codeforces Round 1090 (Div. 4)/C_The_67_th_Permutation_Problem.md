# **【算法题】C. The 67th Permutation Problem（第67个排列问题）**
每测试用例时间限制：2 秒
每测试用例内存限制：256 兆字节

### 题目描述
到学校后，猕猴颇为唐突地被他的朋友AG - 88301迎接，后者因为整晚都在向一个毫无戒心的陌生人喋喋不休地谈论他对考拉兹猜想的开创性证明以及他第67次单相思，而没做家庭作业。所以，按照惯例，AG - 88301带着越来越少的感激之情，让猕猴帮他做作业。此时，猕猴受够了，转而向他的手下（也就是你们！）来解决AG - 88301的作业任务。

给定一个整数 \(n\) 。你必须构造一个长度为 \(3n\) 的排列\(^{*}\) ，使得如果你将这个排列分成 \(n\) 个连续的块，每个块有 \(3\) 个元素，这些块的中位数\(^{\dagger}\)之和最大。

更正式地说，你必须构造一个长度为 \(3n\) 的排列 \(p\) ，使得 \(\sum_{i = 0}^{n - 1} \text{median}(a_{3i + 1}, a_{3i + 2}, a_{3i + 3})\) 最大。如果有多个可能的 \(p\) ，输出任意一个。

### 名词解释
 - \(^{*}\) 长度为 \(n\) 的排列是一个由 \(n\) 个从 \(1\) 到 \(n\) 的不同整数按任意顺序组成的数组。例如，\([2, 3, 1, 5, 4]\) 是一个排列，但 \([1, 2, 2]\) 不是排列（数组中 \(2\) 出现两次），\([1, 3, 4]\) 也不是排列（\(n = 3\) 但数组中有 \(4\) ）。
 - \(^{\dagger}\) 包含 \(3\) 个元素的数组 \(b\) 的中位数是 \(b\) 按非递减顺序排序后的第 \(2\) 个元素。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 \(t\)（\(1 \leq t \leq 10^4\)）。接下来是测试用例的描述。

每个测试用例的第一行也是唯一一行包含一个整数 \(n\)（\(1 \leq n \leq 10^5\)）。

保证所有测试用例中 \(3n\) 的总和不超过 \(3 \cdot 10^5\) 。

### 输出
对于每个测试用例，输出一个排列 \(p\) ，使得连续块的中位数之和最大。如果有多个可能的 \(p\) ，输出任意一个。

### 示例
- **输入**
```
3
2
1
3
```
- **输出**
```
1 3 4 2 5 6 
3 1 2
5 2 4 8 3 9 7 1 6
```

### 说明
在第一个测试用例中，\([1, 3, 4, 2, 5, 6]\) 是一个可能的答案，因为 \(\text{median}(1, 3, 4) + \text{median}(2, 5, 6) = 3 + 5 = 8\) ，并且可以证明 \(8\) 是中位数之和的最大可能值。

在第二个测试用例中，\([3, 1, 2]\) 是一个可能的答案，因为当 \(n = 1\) 时，唯一可得到的中位数之和是 \(2\) 。
