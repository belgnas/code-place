# **【算法题】E. Coloring a Red Black Tree（红黑树着色）**
每个测试用例的时间限制：2 秒
每个测试用例的内存限制：256 兆字节

给定一棵有 \(n\) 个节点的树，节点编号从 \(1\) 到 \(n\) ，以及一个长度为 \(n\) 的二进制字符串 \(s\)，其中 \(s_i = 1\) 表示第 \(i\) 个节点被染成红色，\(s_i = 0\) 表示第 \(i\) 个节点被染成黑色。你可以执行以下操作：
1. 选择一个节点 \(a\)（\(1 \leq a \leq n\)）。
2. 从 \(a\) 的邻居中随机均匀选择一个邻居 \(b\)。
3. 用 \(b\) 的颜色重新给节点 \(a\) 染色。注意，改变颜色的是你选择的节点，而不是随机选择的那个节点。

你希望通过执行任意次数（可能为零次）的操作，将树中的所有节点都染成红色。如果以最优方式执行操作，请计算达到此目标所需的最小期望操作次数。初始时至少有一个红色节点。

### 输入
每个测试包含多个测试用例。第一行包含测试用例的数量 \(t\)（\(1 \leq t \leq 1000\)）。接下来是测试用例的描述。

每个测试用例的第一行包含一个整数 \(n\)（\(1 \leq n \leq 2 \cdot 10^5\)）。

第二行包含一个长度为 \(n\) 的二进制字符串 \(s\)。

接下来的 \(n - 1\) 行每行包含两个整数，第 \(i\) 行包含 \(u_i\) 和 \(v_i\)（\(1 \leq u_i, v_i \leq n\)），表示节点 \(u_i\) 和 \(v_i\) 之间有一条边。

保证给定的图是一棵树，至少有一个红色节点，并且所有测试用例中 \(n\) 的总和不超过 \(2 \cdot 10^5\)。

### 输出
对于每个测试用例，输出一个实数，表示最小期望操作次数。

如果你的答案与标准答案的绝对误差或相对误差不超过 \(10^{-6}\)，则认为你的答案是正确的。

正式地，设你的答案为 \(x\)，评判答案为 \(y\)。当且仅当 \(\frac{|x - y|}{\max(1, |y|)} \leq 10^{-6}\) 时，你的答案将被接受。

### 示例
**输入**
```
3
3
101
1 2
2 3
5
10010
1 2
2 3
3 4
2 5
3
010
1 2
2 3
```
**输出**
```
1.00000000000000000000
4.50000000000000000000
2.00000000000000000000
```

### 说明
在第一个测试用例中，只有一个黑色节点，它只有两个红色邻居。因此，如果我们首先对第二个节点执行操作，只需一次操作就能得到一棵全红的树。
